# Spec: 账号隔离、离线缓存与请求可靠性

## 目标

保证同一块手表切换 QQ/微信账号后不会串用播放快照、最近播放或离线文件；等待 Wi‑Fi 的缓存任务在真正执行时重新签发播放地址，并把音频、封面和歌词组成可离线使用的完整包。

## 规则

- 播放快照必须携带 `ownerAccountId`，只有当前登录账号完全匹配时才能恢复。
- 最近播放以 `(trackId, ownerAccountId)` 为主键；旧版本没有归属信息的记录在数据库迁移时清除。
- 离线文件继续位于账号哈希目录；退出后保留但锁定，只有原账号重新登录后可继续或播放。
- WorkManager 输入只保存歌曲标识与首选音质，播放 URL 在 Worker 获得网络并开始执行时签发；每次重试重新签发。
- 同时运行的下载最多 2 个；完成后尽力缓存封面和歌词，附属文件与音频一起删除。
- 缓存任务入队时立即写入数据库，网络约束未满足时也必须显示“等待 Wi‑Fi/排队中”。
- 开始和写入过程中均需保留“剩余文件大小 + 256MB”空间；退出或换号时取消全部缓存 Worker 并把未完成任务改为暂停。
- 其他账号缓存只显示匿名汇总，但允许经过二次确认后整体删除；当前账号支持按缓存分组整体删除。
- 搜索新请求取消旧请求，旧请求不得覆盖新关键字或新类型的结果。
- 喜欢列表、歌单、专辑和歌手详情必须分页读取，不能在 100 首静默截断。
- 收藏歌单使用独立只读接口合并；所有歌单写目标必须属于当前账号且具有正数目录 ID，缺少歌曲数字 ID 时先读取详情。
- 日志统一隐藏 HTTP(S) 地址、Cookie、访问/刷新令牌和二维码票据。
- 播放恢复只对当前歌曲执行一次；歌曲切换、恢复成功或恢复失败都必须释放重试锁。
- MediaController 未连接时把操作排队到连接完成，不在主线程同步等待。

## 验证

- JVM 单测：账号快照、缓存路径、QRC、搜索/队列纯函数。
- Android 测试：v2→v3 数据库迁移、主界面启动和免责声明交互；Robolectric 在 API 24/36、480×480 环境执行迁移与 Activity 创建测试。
- 交付：`testDebugUnitTest`、`compileDebugAndroidTestKotlin`、`lintRelease`、`assembleRelease`、签名与 SHA-256。
