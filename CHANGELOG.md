# Changelog

## 0.9.5 — 2026-07-18

- 修复 `v0.9.3/v0.9.4` 使用沙箱调试密钥导致的覆盖安装签名不一致，恢复 `v0.9.0–v0.9.2` 的原始证书。
- Release 改为仓库外固定私钥和显式配置，构建前强制校验证书 SHA-256，禁止再回退到默认 debug key。
- 增加签名配置模板、历史兼容说明和 Release 签名审计规范。

## 0.9.4 — 2026-07-17

- 歌词高亮由整字符跳变改为连续进度遮罩，QRC 逐字时间和普通 LRC 相邻行时间均以像素级填充平滑渲染。
- 当前歌词切换时增加淡入、轻微缩放和左移动画，保留点击跳转与屏幕居中跟随。
- 原文和翻译都强制单行显示，时间使用独立窄列；长句按可用宽度动态缩小，极端长度截断但不会换行。

APK SHA-256：`410887f1d2dd6d111890b187bdb6210781481bab5f9ac5580cd5328ebc08a785`

## 0.9.3 — 2026-07-17

- 无时间轴歌词不再显示为空：纯文本原文与翻译按行展示；歌曲没有翻译时自动保留原文，不显示占位提示。
- Android 13+ 增加媒体通知权限；MediaSession 发布上一首/下一首按钮，耳机媒体键与歌曲自然结束均可在无界面时按持久化队列继续播放。
- 会员资料按同一权益记录绑定类型与到期日；同账号缓存未过期时不重复请求，失败后不再永久显示“正在读取”。
- 独立分页读取收藏歌单；搜索游标按原始页大小计算，喜欢/歌单写入前自动补齐歌曲数字 ID，并拒绝无效目录。
- 缓存入队即显示“等待 Wi‑Fi/排队中”，下载前预留文件剩余大小加 256MB；支持整组删除和匿名删除其他账号锁定缓存。
- 二维码主页面增加加载、错误与重试状态；日志补充所有上游凭据别名脱敏，播放质量降级和登录失效给出明确提示。
- “我的”页只在资料尚未准备好时加载，避免每次左右滑动重复请求和大列表重组。

APK SHA-256：`de42bf203d6b30dd2714f071b418e6e43973467d4804e1c661cdb654119b475e`

## 0.9.2 — 2026-07-16

- 修复封面横向手势抢占分页：播放器左滑始终进入歌词页，封面保留双击播放/暂停和上下滑音量。
- 登录渠道以 QQ 音乐返回的 `tmeLoginType` 为准；新登录会清理旧账号内存态和 WebView Cookie。
- 喜欢列表、歌单、专辑和歌手详情支持超过 100 首的分页读取。
- 登录后首页优先加载账号个性化推荐，数量不足时使用新歌补齐“换一换”批次。
- 搜索兼容新版嵌套音质字段，320k、媒体 MID、专辑和 VIP 标识不再丢失。
- 缓存进度显示已下载/总大小和百分比；暂停临界点保留已完成文件，封面使用临时文件原子落盘。
- 直接重新登录时会停止旧账号播放、取消未完成请求并清空内存队列，异步结果不会写入新账号。
- 详情页补回明确的返回与整页缓存入口，歌曲可选择加入指定的个人歌单；其他账号缓存只显示匿名锁定汇总。
- 日志统一隐藏播放 URL、Cookie、令牌和二维码票据；扫码桥接限制官方来源和消息格式。

APK SHA-256：`970a572b62bd69ecf3acef25523c139ddf281ed25754e14566c35409b6228047`

## 0.9.1 — 2026-07-16

- MediaController 连接失败或被取消时，播放状态、定时关闭和控制操作安全降级，不再因 `Future.get()` 抛异常而崩溃。
- 暂停或删除缓存前等待 WorkManager 确认任务已停止，避免文件或数据库记录在删除后被后台任务写回。
- 缓存控制异常统一写入日志并反馈到界面，不再成为未捕获协程异常。
- 加入 Robolectric 运行测试，在 API 24 与 API 36 的 480×480 环境验证 Activity 创建，并实际验证 v2→v3 Room 数据库迁移。

## 0.9.0 — 2026-07-16

- 播放快照、最近播放和离线文件严格按账号隔离，退出后缓存保留但锁定。
- 缓存任务在实际执行时重新签发播放 URL，最多并发 2 个，并同时保存封面和歌词。
- 修复播放恢复锁、MediaController 主线程等待、搜索结果竞态和歌词失败后无法重试。
- 歌单删除增加确认；歌曲加入歌单时明确选择目标，不再默认写入任意歌单。
- 使用 MIT 许可实现 QQ QRC 解密和精确逐字时间，异常时自动回退普通 LRC。
- 加入 v2→v3 数据库迁移测试、方屏界面烟雾测试及 GitHub Release 更新检查回退。
