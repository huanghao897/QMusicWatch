# Changelog

## 0.9.1 — 2026-07-16

- MediaController 连接失败或被取消时，播放状态、定时关闭和控制操作安全降级，不再因 `Future.get()` 抛异常而崩溃。
- 暂停或删除缓存前等待 WorkManager 确认任务已停止，避免文件或数据库记录在删除后被后台任务写回。
- 缓存控制异常统一写入日志并反馈到界面，不再成为未捕获协程异常。
- 加入 Robolectric 运行测试，在 API 24 与 API 36 的 480×480 环境验证 Activity 创建，并实际验证 v2→v3 Room 数据库迁移。

## 0.9.0 — 2026-07-16

- 播放快照、最近播放和离线文件严格按账号隔离，退出后缓存保留但锁定。
- 缓存任务在实际执行时重新签发播放 URL，最多并发 2 个，并同时保存封面和歌词。
- 修复播放恢复锁、MediaController 主线程等待、搜索结果竞态和歌词失败后无法重试。
- 歌单删除增加确认；歌曲加入歌单时明确选择目标，不再默认写入任意歌单。
- 使用 MIT 许可实现 QQ QRC 解密和精确逐字时间，异常时自动回退普通 LRC。
- 加入 v2→v3 数据库迁移测试、方屏界面烟雾测试及 GitHub Release 更新检查回退。
